## 背景与目标

- 当前 CLI 直接在本地构建上下文并进入交互循环，不符合“CLI 是 Web 包装层”的新口径。
- 本子需求要把 CLI 改造成两种模式都通过 Web 请求执行的统一入口。

## 当前口径

1. CLI 必须同时支持交互式模式与带参数单次执行模式。
2. 两种模式都必须通过 Web 请求完成执行，而不是直接在 CLI 进程里执行业务逻辑。
3. 单次执行模式成功后输出到 stdout 并退出。
4. CLI 的可安装命令名固定为 amiyabot-cli，并作为默认调用入口写入文档和交付说明。

- 当前状态：`todo`

## 功能拆解 / 实施拆解

1. 设计 CLI 启动参数与模式切换规则。
2. 将交互模式改造成基于 Web 请求的循环。
3. 新增带参数单次执行模式。
4. 校准帮助信息、错误输出与退出行为。

## 验收标准

1. CLI 同时支持交互式模式与带参数单次执行模式。
2. 两种模式都通过 Web 接口完成执行。
3. 单次执行结果输出到 stdout 后退出，并具有明确退出码语义。
4. 安装后应生成 amiyabot-cli 命令，并作为默认 CLI 入口。
