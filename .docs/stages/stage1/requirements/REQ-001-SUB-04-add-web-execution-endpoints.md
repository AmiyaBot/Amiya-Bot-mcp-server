## 背景与目标

- 当前 Web 侧只有状态接口，没有 CLI 可调用的执行接口。
- 本子需求的目标是把 Web 服务补齐为统一执行核心可调用的对外接口面。

## 当前口径

1. 执行接口必须复用 `SUB-03` 的共享服务层。
2. 探活接口继续保留，供 CLI 在本地模式下做服务可用性检查。
3. 本子需求只处理 Web 侧接口，不直接改 CLI。

## 当前状态视图

- 当前状态：`todo`
- 当前目标：在 FastAPI 侧补出稳定可调用的执行接口。
- 当前已知进展：现有 `/rest/status` 可继续作为探活基线。
- 下一步动作：
  1. 根据 `SUB-01` 契约设计执行路由。
  2. 将执行路由接到共享服务层。
  3. 验证探活与执行接口行为。

## 功能拆解 / 实施拆解

1. 固定执行接口路径与请求模型。
2. 将执行请求映射到共享服务层。
3. 保持 `/rest/status` 的探活作用清晰可用。
4. 输出 Web 执行接口的回归验证记录。

## 验收标准

1. Web 服务提供 CLI 可调用的稳定执行接口。
2. 执行接口行为与 `SUB-01` 契约一致。
3. `/rest/status` 或等效探活接口可稳定用于 CLI 探活。
