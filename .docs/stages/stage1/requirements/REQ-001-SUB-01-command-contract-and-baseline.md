## 背景与目标

- 当前 CLI 直接在本地进程内构建上下文并执行命令，Web 侧只有状态接口，尚未形成统一执行契约。
- 后续的 Web 执行入口、CLI 包装化与本地自启动都依赖这里冻结统一口径。

## 当前口径

1. 共享执行接口已固定为 `POST /rest/commands/execute`，请求体包含 `command` 与 `args`。
2. 默认命令服务地址已固定为 `http://127.0.0.1:9000/`，并通过 `--url` 或 `CommandServiceUrl` 覆盖。
3. 本地 URL 判定范围已固定包含 `127.0.0.1`、`localhost`、`::1`、`0.0.0.0`；探活接口固定为 `/rest/status`。
4. 单次执行模式以 `ok + output` 作为统一响应语义，CLI 按返回结果映射 stdout 与退出码。
